Arkime 4.0

Cont3xt, Role Permissions, Hide Graph/Map, Speed Up Initial Load, Wildcard Shortcuts, Bug Fixes and More

We are excited to share that Arkime 4.0 is now available! This release includes an entirely new web application (yay Cont3xt!), a new permission model with roles, speedier queries by hiding or disabling the graph and map, new AND arrays with ][ syntax vs OR arrays with [], speedier initial load time by combining multiple calls from UI into one, shortcut wildcard support, bug fixes, and much, much more. View a list of all the changes here.

Learn how to upgrade to 4.0 now!

Breaking Changes

  • Must be on v3.3.0+ to upgrade to 4.x
  • Elasticsearch before 7.10 is not supported
  • Requires NodeJS 16.x
  • Use roles for permission checking - userAdmin role required to edit users.
    addUser.js - new --roles option, --admin creates superAdmin user
  • In header auth mode, userAuthIps allows only localhost by default
  • Non-standard pcap files now use the .arkime extension
  • WISE multiES prefix now default to arkime_
  • WISE threatstream source require a manually created md5 index
  • the version file lives in common directory now
  • new defaults maxFileSizeG=12, compressES=true
  • pcap compression is turned on by default with simpleCompression=none
  • right-click changed to value-actions in config

Cont3xt

Cont3xt intends to centralize and simplify a structured approach to gathering contextual intelligence in support of technical investigations. It enriches indicators using commercial and OSINT sources in a structured, consistent, and thorough process. Learn more here!

Role Permissions

Arkime has moved to a role permission model to check for user access to resources. Each user can be assigned a list of roles. You can create/update/delete roles from the Arkime or Cont3xt users page. View default Arkime roles here.

Arkime Shortcuts, Notifiers, Periodic Queries, Views, and Hunts can now be shared via Arkime roles or to specific users.

Hide/Disable Graph/Map

Fetching the data to compute the graph and map takes much longer than a general Arkime query for sessions. By hiding both the graph and the map, you can easily speed up session searches. This is particularly useful when issuing queries for long periods of time. A user can manually hide the graph and map and each cluster can be configured with a maximum number of days to auto-disable the graph and map (which can be overridden per user request).

Speed Up Initial Load

Initially loading the Arkime web application requires a lot of data. To improve the speed of the initial load time many queries have been combined into one, here. Also, some resources are now lazy loaded by waiting for the user to request their presence before loading (e.g. the graph and map libraries).

Wildcard Shortcuts

You can now issue queries for many shortcuts at a time by using a wildcard like this: ip.src == $TEST_*.
